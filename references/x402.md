# x402 Pay-Per-Verify

## Overview

UZPROOF's `/api/verify` endpoint is gated behind the [x402 HTTP payment protocol](https://www.x402.org/), **live on Solana mainnet since 2026-04-16**. AI agents and B2B backends pay per request with USDC ‚Äî no API keys, no signup.

**Current status:** live in production. `X402_ENABLED=true` on `uzproof.com`. First real payment validated 2026-04-16.

## How It Works

1. Agent sends `POST /api/verify` without an `X-Payment` header
2. UZPROOF returns HTTP `402 Payment Required` with x402 payment schema
3. Agent signs a USDC transfer for `maxAmountRequired` to `payTo` and broadcasts it
4. Agent retries the same request with `X-Payment: <tx signature>` ‚Äî UZPROOF validates the payment on-chain and returns the verification result

One signature = one verify call. Replays are rejected (Redis atomic `SET NX` replay guard). Payments older than **5 minutes** are rejected as stale ‚Äî sign, broadcast, call within that window.

## Pricing

| Operation | Cost | Endpoint |
|---|---|---|
| `verify()` | **$0.05 USDC** | `POST /api/verify` |
| `getAttestation()` | Free | `GET /api/sas/status` |
| `detectContract()` | Free | `GET /api/contracts/detect` |
| `getTokenInfo()` | Free | `GET /api/tokens/info` |

SAS attestations live on-chain and any caller can read them directly via RPC, so charging for our convenience wrapper just pushes integrators to bypass us. Verify is the paid surface.

## Integration ‚Äî SDK (recommended)

The `@uzproof/verify` SDK (‚â• 1.2.0) exposes `PaymentRequiredError` and an `xPayment` option:

```typescript
import { UzproofClient, PaymentRequiredError } from '@uzproof/verify';

const client = new UzproofClient();

async function verifyWithX402(
  wallet: string,
  action: string,
  config: object,
  payForCall: (amountUsdc: string, payTo: string) => Promise<string>, // returns tx signature
) {
  try {
    // First attempt ‚Äî will 402
    return await client.verify({ wallet, action: action as any, config });
  } catch (err) {
    if (err instanceof PaymentRequiredError) {
      const scheme = (err.details as any).schemes[0];
      const sig = await payForCall(scheme.maxAmountRequired, scheme.payTo);
      return await client.verify(
        { wallet, action: action as any, config },
        { xPayment: sig },
      );
    }
    throw err;
  }
}
```

## Integration ‚Äî Direct HTTP

```typescript
async function verifyWithX402(wallet: string, action: string, config: object) {
  // Step 1: Probe for payment schema
  const res = await fetch('https://uzproof.com/api/verify', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ wallet, taskType: action, config }),
  });

  if (res.status === 402) {
    const paymentInfo = await res.json();
    // paymentInfo.schemes[0] = { scheme, network, maxAmountRequired, payTo, token, ... }
    const scheme = paymentInfo.schemes[0];

    // Step 2: Pay USDC on Solana mainnet
    // (use @solana/web3.js + @solana/spl-token to send `maxAmountRequired`
    //  USDC from your signer wallet to scheme.payTo)
    const paymentSig = await sendUsdcTransfer(scheme.payTo, scheme.maxAmountRequired);

    // Step 3: Retry with X-Payment header
    const retryRes = await fetch('https://uzproof.com/api/verify', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'X-Payment': paymentSig,
      },
      body: JSON.stringify({ wallet, taskType: action, config }),
    });
    return retryRes.json();
  }

  return res.json();
}
```

## Important Constraints

- **Mode restriction.** x402 callers must use **direct mode** (`taskType` + `config`). Passing `taskId` returns `400 DB_MODE_SELF_ONLY` ‚Äî we don't let paying callers advance someone else's quest progress for $0.05.
- **Rate limit.** x402 tier: 60 calls/minute per caller IP (self-auth tier: 20/min). Per-payment-wallet limits land in Phase 2.
- **Freshness window.** Payments older than 5 minutes are rejected. Sign-broadcast-call tightly.
- **Overpayment.** Sending more than `maxAmountRequired` is accepted (treated as a tip). You don't get extra calls.
- **Read-only endpoints are free.** `getAttestation`, `detectContract`, `getTokenInfo` never charge.

## Error Codes Callers Should Handle

| HTTP | error_code | Meaning |
|---|---|---|
| 402 | ‚Äî (body carries schema) | Payment required ‚Äî pay and retry with `X-Payment` |
| 401 | `AUTH_REQUIRED` | No session cookie and no payment |
| 400 | `DB_MODE_SELF_ONLY` | x402 caller used `taskId` ‚Äî switch to `taskType` + `config` |
| 400 | `VAL_INVALID_REQUEST` | Malformed wallet or unsupported action type |
| 429 | `RATE_LIMITED` | Exceeded 60/min (x402) or 20/min (self) |
| 500 | `SRV_INTERNAL_ERROR` | Server-side ‚Äî retry with backoff |

## Why x402 for Verification?

- **No API keys.** Agents pay per use, no signup. Perfect for autonomous agents that come and go.
- **Micro-payments.** $0.05 per call is viable for high-volume use (Jupiter-scale batch jobs: ~1200/hr headroom per IP).
- **Composable.** Works with any x402-compatible client (x402-proxy, OpenDexter, SP3ND, QuickNode).
- **USDC on Solana.** Stablecoin, ~$0.00025 tx fee, 400ms finality. Paying for verification doesn't eat into margins.
- **Open standard.** Published by Coinbase as an open HTTP spec ‚Äî not proprietary.
